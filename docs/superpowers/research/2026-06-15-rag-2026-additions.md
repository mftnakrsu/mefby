# What to add to the RAG series — 2026 deep dive

> Deep-research output (2026-06-15). 95 agents, 3 search angles, 15 sources fetched,
> 73 claims extracted → 25 adversarially verified (2/3 refutes to kill): **22 confirmed,
> 3 killed**. Lens: genuinely-new-vs-the-12-parts + framework-free/offline/by-hand-teachable.
> Sources are primary (ICLR/NAACL/SIGIR/ICML/EMNLP + Anthropic cookbook) unless noted.

## Tier 1 — strongest, genuinely new, build-by-hand (recommended new parts)

### A. Late-interaction retrieval — ColBERT → ColBERTv2 → ColPali/ColQwen  ★ top pick
- **What:** token-level **multi-vector** embeddings scored by **MaxSim** (for each query
  token, take the max similarity over all doc tokens, then sum) instead of one pooled dense
  vector. Doc vectors are **precomputed offline**.
- **Why 2026:** the headline retrieval-quality paradigm between single-vector dense (Part 7)
  and cross-encoder reranking (Part 8); **ColPali/ColQwen (2024–25, ICLR 2025)** extend it to
  *document images* — a VLM embeds page images directly (~1024 patch vectors), bypassing
  OCR/chunking. Distinct from Part 10's one-line multimodal mention.
- **Evidence:** high. ColBERT SIGIR 2020 (arxiv 2004.12832), ColBERTv2 NAACL 2022
  (2112.01488), ColPali ICLR 2025 (2407.01449). In-repo grep for colbert/maxsim/multi-vector = 0 hits.
- **By-hand:** MaxSim is ~10 lines of numpy; the storage/accuracy tradeoff (per-token vectors
  cost more even at 1–2 bit; ColBERTv2 cuts MS MARCO 154GB→16–25GB) is trivial arithmetic.
  **Fidelity caveat:** ColPali needs a real VLM — teach the *mechanism* (MaxSim, patch vectors)
  with a transparent toy multi-vector stand-in, same as the series' fallback pattern.

### B. Context-aware chunking — late chunking + Anthropic Contextual Retrieval
- **What:** two **training-free** ways to stop chunks from losing their document context.
  - *Late chunking* (Jina, arxiv 2409.04701): embed ALL tokens of a long doc first, then pool
    token spans into chunks **after** the transformer — so "she" still resolves to "Alice".
  - *Contextual Retrieval* (Anthropic): prepend an LLM-generated per-chunk situating sentence
    **before** embedding.
- **Why 2026:** both are cheap, drop-in retrieval-quality wins. Contextual Embeddings alone cut
  top-20 retrieval failure **35%** (5.7→3.7%).
- **Evidence:** high. ⚠️ The widely-quoted **67% / 47%** additive figures were **REFUTED (0-3)** —
  cite only the **35%** alone.
- **By-hand:** ~30 lines each. Late chunking needs a long-context mean-pooling encoder (teach the
  span-pooling mechanism with a toy encoder); contextual retrieval's "prepend-then-embed" runs
  with the series' offline generate() stand-in. Distinct from Part 5 (static title prepend) and
  Part 9 (compression).

### C. Adaptive RAG — route each query by complexity
- **What:** a small classifier routes each query to **no-retrieval / single-step / multi-step**
  retrieval (don't pay for retrieval the query doesn't need; don't under-retrieve hard ones).
- **Why 2026:** an emerging best practice; ties the whole series together (route among the
  pipelines built in Parts 6–10). Distinct from Part 8 (query transforms) and Part 10 (routing
  by *source*, not complexity).
- **Evidence:** high. Adaptive-RAG NAACL 2024 (arxiv 2403.14403), public code, LangGraph/LlamaIndex
  tutorials. (~35% latency / ~28% cost cuts are vendor/indicative.)
- **By-hand:** a tiny rule/keyword complexity classifier dispatching to existing pipelines — fully
  offline.

## Tier 2 — eval-flavored lessons (timely, slightly newer)

### D. Long-context vs RAG, head-to-head  (honest answer to the "RAG is dead" hype)
- **What:** LLM-alone (stuff everything in context) vs top-k retrieval, compared on
  accuracy/cost/latency over a **synthetic fictional corpus + inserted needles** (Starlight
  Academy / U-NIAH) so pretrained knowledge can't leak the answer.
- **Evidence:** high. U-NIAH ACM TOIS (2503.00353), Self-Route EMNLP 2024 (2407.16833), LaRA
  ICML 2025. Modest framing: the *debate* is real — **not** "RAG is dead".
- **By-hand:** pure numpy, substring-match scoring. Distinct from Part 11 faithfulness. A great,
  myth-busting addition.

### E. Capability/step-level agentic-RAG evaluation (RAGCap-Bench)  — frontier sidebar
- **What:** decompose agentic RAG into intermediate capabilities (planning, evidence extraction,
  grounded reasoning, noise robustness); intermediate scores predict end-to-end QA.
- **Evidence:** **medium** (single source 2510.13910, 3 same-family models, 255 MCQs, very recent).
  Good as a *frontier sidebar* in Part 11, not settled canon.

## Tier 3 — lower priority / orthogonal

- **Cross-session agent memory** (write-side ops: promotion, consolidation, decay; Redis Agent
  Memory preview, open-source Hindsight, MIT). High-confidence it exists, but novelty is
  **orthogonal to retrieval** — probably a separate "agents" series, not core RAG.
- **CSR adaptive sparse embeddings** (ICML 2025 oral, 2503.01776) — alternative to Matryoshka.
  **Medium** (single-lineage, unreplicated). Refines Part 2 embeddings, not a new stage.
